## 第4页 — 研究背景: 化工AI的多尺度挑战

**标题**: 化工领域的天然多尺度属性

**叙事逻辑**: 化学工程本质上是一个多尺度学科——从原子间的化学键到数千公里管线，每个尺度都有独立的AI工具，但彼此孤立。

```
化学工程的多尺度谱系:

  原子尺度              分子/反应尺度          过程/设备尺度          工厂/系统尺度
  (Å ~ nm)             (nm ~ μm)             (m ~ 10²m)            (10²m ~ km)
  ─────────            ────────────          ────────────          ────────────
  晶体结构              化学反应              单元操作              全厂流程
  电子结构              热力学性质              换热网络              供应链
  DFT计算               动力学模型              PFD / P&ID           数字孪生
  
  ↑ 材料科学的           ↑ 传统化工             ↑ 过程系统工程          ↑ 工业工程
    主战场                 核心领域               核心领域               核心领域
  
  现状: 各尺度AI工具繁荣但孤立
  · 原子尺度: GNN/Diffusion 做晶体生成 (如 crysllmgen)
  · 分子尺度: LLM 做化学知识问答 (如 ChemELLM)
  · 过程尺度: 图神经网络做流程模拟
  · 工厂尺度: 强化学习做调度优化
  
  问题: 四个尺度各自为政, 跨尺度关联靠人工经验, 无统一框架
```

## 第5页 — 研究动机: 为什么要跨尺度?

**标题**: 局部最优 ≠ 全局最优

```
核心论点: 单一尺度的优化可能被其他尺度的约束所抵消

案例1 — 催化剂开发 (原子→分子→过程):
  原子尺度: DFT找到高活性催化剂, 活性+15%
  分子尺度: 但该催化剂在反应温度下快速失活
  过程尺度: 且反应放热增大, 现有换热器无法移除额外热量
  → 全厂效益反而下降 3%

案例2 — 晶体材料发现 (原子→宏观):
  原子尺度: crysllmgen 生成新晶体结构, valid rate 99.9%
  宏观尺度: 但该结构无法大规模合成, 或对湿度/氧气敏感
  → 停留在论文里, 无法工业化

结论: AI4S 的真正价值不在单点突破, 而在跨尺度理解与协同优化
```

**视觉**: 两个案例各一个流程图, 展示尺度间的约束传递

## 第8页 — 核心科学问题 (重写: 多尺度叙事)

**标题**: 核心科学问题: 如何建立跨尺度的统一AI框架?

```
现有方法仅针对单一尺度的化工问题——单模态知识问答(分子尺度),
纯结构生成(原子尺度), 或独立流程建模(过程尺度)——无法建模跨尺度关联。

三个核心科学问题:

1. 跨尺度表示问题:
   不同尺度的数据结构完全不同——
   原子尺度: 3D晶体坐标图 (连续, 周期性)
   分子尺度: 自然语言 + 化学反应式 (离散, 符号化)
   过程尺度: PFD/P&ID拓扑图 (异构, 符号+连续混合)
   工厂尺度: 时序数据 + 优化目标 (动态, 数值)
   → 是否存在统一的表示空间? 如何对齐?

2. 物理一致性约束问题:
   跨尺度推理必须满足守恒律 (质量/能量/动量)
   但现有AI模型不理解物理约束
   → 如何将物理先验嵌入模型架构与损失函数?

3. 闭环验证问题:
   原子尺度预测 (晶体结构) → 需要宏观尺度验证 (合成+测试)
   AI生成 ≠ 物理可实现
   → 如何建立从原子到宏观的验证闭环?

因此, 我的研究从四个尺度分别入手, 最终目标为跨尺度统一:
  原子尺度: 晶体材料生成 (crysllmgen — 已发表 NeurIPS 2025)
  分子尺度: 化工大模型平台 (ChemELLM — BESC 2025)
  过程尺度: 多模态过程知识图谱 (博一)
  跨尺度统一基础模型 (博士核心)
```

**视觉**: 四层尺度金字塔 + 科学问题标注

## 第11页 — 原子尺度: crysllmgen 晶体材料生成

**标题**: 原子尺度: LLM + 扩散模型的混合晶体生成框架

**叙事**: 在多尺度框架的最底层——原子尺度, 我提出了一种混合框架来生成晶体材料。核心洞察: LLM擅长捕捉化学组成规律但缺乏精确坐标, 扩散模型擅长精确坐标生成但缺乏化学先验——两者互补。

```
crysllmgen: LLM Meets Diffusion — NeurIPS 2025

两阶段流水线:

  Stage 1: LLM 成分生成
    LLaMA-2/3 + LoRA 微调
    输入: 文本提示 (空间群/元素/性质要求)
    输出: 离散晶体属性 (原子种类, 分数坐标, 晶格参数)
    → 结构有效但坐标粗糙 (valid ~92%)

  Stage 2: 扩散坐标精修
    CSPDiffusion (等变GNN骨干CSPNet)
    PC Sampling (Predictor-Corrector, 1000步)
    输入: LLM生成的粗糙结构作为 x_T
    输出: 精修后的连续坐标+晶格
    → 结构有效性提升至 ~99.9%

关键设计选择:
  · 解耦范式: LLM定成分, 扩散定结构——职责分明
  · 消融实验表明: 在扩散过程中注入LLM特征反而损害覆盖率 (-46%)
    → LLM的价值在于"初始化", 而非"引导"
  · 扩散无条件精修效果最优: COV-Recall 99.15%, COV-Precision 99.18%
```

**视觉**: 两阶段流水线图 + 消融实验结果表格

## 第12页 — 原子到宏观: 晶体发现的尺度跨越

**标题**: 从原子尺度到宏观尺度: 晶体材料的验证闭环

**叙事**: 原子尺度生成的结构 (crysllmgen输出) 必须经过宏观尺度的验证才能确认其价值。这体现了多尺度框架的必要性——原子尺度的成功不等同于材料发现的成功。

```
原子尺度 → 宏观尺度的验证链:

  原子尺度 (crysllmgen)           宏观尺度验证
  ┌──────────────────┐         ┌──────────────────┐
  │ LLM生成成分+粗结构 │         │ DFT能量计算       │
  │ ↓                │         │ · 形成能          │
  │ 扩散精修坐标      │  ────→  │ · 带隙            │
  │ ↓                │         │ · 弹性常数        │
  │ 输出: CIF结构     │         │ · 声子谱 (稳定性)  │
  └──────────────────┘         └────────┬─────────┘
                                        │ 通过DFT验证
                                        ↓
                               ┌──────────────────┐
                               │ 湿实验合成+表征    │
                               │ · XRD 结构确认    │
                               │ · SEM/TEM 形貌    │
                               │ · 目标性能测试     │
                               └──────────────────┘

多尺度视角的意义:
  · 原子尺度: crysllmgen 解决"能否存在" (热力学稳定相)
  · 宏观尺度: DFT+实验解决"能否合成+能否用" (动力学可达+性能达标)
  · 两者的断裂正是当前AI材料发现的瓶颈——valid 99.9% ≠ 可合成
```

**视觉**: 从左到右的验证链, 标注尺度跨越

## 第19页 — 跨尺度统一基础模型

**标题**: 跨尺度统一基础模型 (博士核心)

(内容与v5基本一致, 扩展多尺度对齐细节)

```
问题: 原子尺度(crysllmgen+DFT)、分子尺度(ChemELLM)、过程尺度(PFD+P&ID)
      各自独立, 缺乏统一框架

目标: 建立跨尺度统一基础模型——类似于为化工AI建立"统一场论"

核心假设:
  存在一个统一的表示空间, 可同时编码:
  · 原子尺度: 晶体结构 (3D坐标图, crysllmgen latent)
  · 分子尺度: 化学反应+文本知识 (ChemELLM embedding)
  · 过程尺度: P&ID拓扑图 (GNN node/edge embedding)
  · 工厂尺度: 时序运行数据

技术路线:
  ① 多模态编码器:
     EquiformerV2 (晶体, SO(3)等变)
     → 通用GNN (P&ID拓扑)
     → LLM (ChemELLM, 文本+反应)
     → 共享投影层映射到统一空间

  ② 跨尺度对比学习:
     正样本: 同一化工体系的不同尺度描述 (如某催化剂↔该反应器↔该工厂)
     负样本: 随机配对的跨尺度数据
     目标: 拉近相关跨尺度表示

  ③ 物理一致性约束:
     跨尺度推理必须满足质量/能量守恒
     → 物理约束作为正则化项嵌入损失函数
     → 违反守恒律的推理被惩罚

  ④ 跨尺度指令微调:
     "该催化剂酸性增强 → 反应器温度应如何调整?"
     "上游PFD中增加换热器 → 下游P&ID中哪些管线需要修改?"

关键挑战:
  · 跨尺度表示的语义鸿沟 (几何的 vs 符号的 vs 拓扑的)
  · 物理约束的可微分嵌入
  · 跨尺度训练数据的构建 (对齐标注成本极高)
```

## 预设问题 (更新)

1. **为什么用多尺度框架组织?** → 化学工程天然是多尺度的——从化学键到工厂。每个尺度的AI工具都在蓬勃发展, 但彼此孤立。我的研究从各尺度分别突破, 最终目标为统一。

2. **crysllmgen 在多尺度中的位置?** → 原子尺度。它解决"能否存在"的问题——生成热力学稳定的晶体结构。但"能否合成""能否用"需要跨越到宏观尺度验证。

3. **四步之间的关系?** → 四步对应四个尺度, 原子(晶体)→分子(反应/知识)→过程(设备/流程)→跨尺度统一。下层为上层提供输入与约束。

4. **跨尺度最大难点?** → 表示空间的异构性: 晶体坐标(连续几何) vs 化学反应(符号) vs P&ID(拓扑图) vs 文本(NL)。需多模态对比学习对齐。

5. **和ChemELLM核心论文(催化学报)的关系?** → 催化学报论文是模型+评测, BESC是系统平台+工程部署。二者服务于分子尺度。
